Paramètres de la page

Seul un titre est requis, mais il est préférable de définir les paramètres suivants au minimum:

title
Titre de la page web
description
Description de la page
lang
Langue de la page courant en format ISO, tel que "en", "fr".
altLangPage
Lien vers la page alternative dans l'autre langue officiel.
dateModified
Date de modification de la page en format ISO, AAAA-MM-JJ.

Si le paramètre altLangPage est omis, le liens vers la page alternative ne sera pas ajouter dans le coin supérieur droit.

Définir le type de gabarit (layout) à utiliser.

layout: default
Gabarit de base et utilisé majoritairement pour la création de page de contenu. Ce paramètre est requis lorsqu'il n'est pas défini comme paramètre par défaut dans la configuration du site. (_config.yml)
layout: fluid
Gabarit de base mais avec un contenu fluide. Toute la largeur de la page sera utilisable.
layout: no-container
Gabarit de base mais avec aucun conteneur de défini ni de titre de niveau 1. Ce gabarit assume que le titre de niveau 1 ainsi que l'une ou l'autre des class CSS délimitant le contenu sera ajouté par l'éditeur web. Les classes CSS de délimitation de contenu possible sont: container et container-fluidun contenu fluide.
layout: without-h1
Gabarit de base mais avec sans titre de niveau 1. L'éditeur web devra l'ajouté directement dans le contenu de la page web.

Les paramètres suivant sont optionnel mais disponible au besoin

titleH1
Si définie, ce texte sera utilisé pour le titre du niveau 1 de la page et l'autre paramètre title sera utilisé pour nommé la page web.
nositesearch
Valeur Booléenne, lorsqu'ajouter avec une valeur "vrai", la recherche de la page, en haut à droite, ne sera pas ajouté.
nomenu
Valeur Booléenne, lorsqu'ajouter avec une valeur "vrai", le menu prinicipal ne sera pas ajouté.
pageType
Définie le type de la page, s'il contient la valeur "archived", la bannière d'archive sera ajouté à la page. (Note technique: prendre en considération d'utilisé une propriété nommé "state" à la place)
breadcrumbs

Liste contenant un item du fil d'ariane. Chaque item est composé d'un objet avec les propriétés suivantes:

link
Liens de l'item du fil d'ariane
title
Titre de l'item du fil d'ariane

L'ordre du fil d'ariane configuré vas du parents vers l'enfant.

Language de codification de la page

Les paramètres de la page (Front-matter) peut être codifié en format YML ou en format JSON.

Le contenu de la page peut être codé avec en mark down ou en html. Le language du contenu est définie par l'extension du fichier éditié. Par example, l'extension .md indique que le contenu de la page est en format "mark down" et l'extension .html indique que le contenu de la page est en format "HTML".
